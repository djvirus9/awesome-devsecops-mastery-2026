# Phase 04: DAST and API Testing

## Goals

- Validate running applications in staging
- Catch runtime misconfigurations and API flaws

## Practices

- Baseline DAST scans in staging
- API testing from OpenAPI specs
- Risk-based triage and follow-up

## Tools

- DAST: [OWASP ZAP](https://www.zaproxy.org/), [Burp Suite](https://portswigger.net/burp)
- API testing: [42Crunch](https://42crunch.com/), [OpenAPI](https://www.openapis.org/)
- Templates: [Nuclei](https://github.com/projectdiscovery/nuclei)

## Deliverables

- Staging scan reports
- API coverage results
- Findings triage workflow
