## Contributing

Thanks for your interest in improving this repository. Please keep changes
focused, documented, and easy to review.

### Quick start

1. Create a branch from `main`.
2. Keep commits small and descriptive.
3. Update or add docs/tests when behavior changes.
4. Open a pull request and complete the checklist.

### Pull request checklist

- [ ] Summary describes the change and why it matters
- [ ] Docs updated when applicable
- [ ] Tests added/updated when applicable
- [ ] Security impact reviewed and noted

### Code of conduct

Be respectful. Disagreement is fine; disrespect is not.
